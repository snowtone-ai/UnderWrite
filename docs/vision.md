# docs/vision.md — UnderWrite

## Purpose
築古住宅（空き家・中古戸建）の買取再販・リフォーム業者に対し、スマホで撮った写真と住所という
最小入力から「買っていい値段（買付上限価格）・再生コストの確率分布（P10/P50/P90）・隠れ損傷リスク・
再販価格・粗利」を即日算出する、アンダーライティング（目利き）AI。

思想: **「開けなくても、わかる。」** 計測アプリでも積算ソフトでもなく、"目利きそのもの（賭けに勝つ判断）"
を売る。点推定ではなく確率分布と買付上限価格を返し、最終判断は事業者に委ねる。

詳細な事業背景・市場・競合は `企画書_UnderWrite_築古住宅アンダーライティングAI.md` を参照（これが一次資料）。

## Target Users
- 一次ニッチ: 北関東〜首都圏近郊で戸建買取再販を年間10〜100戸行う中堅業者（推定約300社）。
- ペルソナ: 仕入れ判断が社長の属人目利きに依存（佐藤社長52歳）／積算・工事管理担当（田中部長44歳）／
  買取再販へ新規参入検討のリフォーム業者（鈴木代表38歳）。

## Success Criteria
- **MVP (Milestone 1) 合格条件**: 実在する築古物件1件を、住所＋写真5〜10枚の入力から
  ダッシュボード出力（買付上限価格・再生コストP10/P50/P90・リスクフラグ・再販価格・粗利）まで
  本番URLで通し、既知の答えと突き合わせて買付上限価格が妥当と確認できること。
- 査定の限界費用が「5万円・2週間」→「数百円・15分」の桁で圧縮される構造であること。
- 同一入力に対し価格出力が決定論的に再現し、回帰テスト可能であること。

## Primary Flows
1. 業者が物件住所を入力し、気になる箇所を含む写真を数枚〜十数枚アップロード。
2. 写真を1枚ずつ非同期でVLM解析 → 構造化された劣化所見（Finding）を生成・DB永続。
3. 住所から公開データ（国交省 不動産情報ライブラリ 等）を照会 → 周辺成約・再販ベースライン取得。
4. 純TypeScriptのアンダーライティングエンジンが Finding＋公開データ＋築年×工法×地域から
   再生コスト確率分布・買付上限価格・粗利を決定論的に算出。
5. 判断ダッシュボードに出力。（将来）施工後の実損傷を入力しモデルを校正（データモート）。

## Non-Goals (現時点)
- ネイティブ iOS RoomPlan / LiDAR 3D復元アプリ（後フェーズ。MVPは素のWeb写真アップロード）。
- PWA / オフライン対応。
- 認証・複数ユーザー・課金（Milestone 1では単一利用者。ハードニングで追加）。
- PDF帳票・工事指示書生成・ハザードマップ・路線価（Milestone 2以降）。
- $0（Claude Pro以外）を超える有料インフラ。無料枠（Vercel / Supabase / Gemini）のみ。

## Failure Cases to Handle
- 写真が不鮮明／枚数不足／無関係画像 → 低信頼度として明示し、過小評価を避ける。
- 公開データAPIの応答欠損・仕様変更 → 型付きクライアントで graceful degrade、キャッシュ利用。
- AIモデル出力がスキーマ非適合 → 境界で zod 検証し reject/retry。生出力を必ず永続し再現可能に。
- モデル差し替え（Gemini→Claude）で価格が変わってはならない（数学は純TSエンジンに隔離）。

## Architectural North Stars
- AIは差し替え可能。全モデル呼び出しは `lib/ai/`（AIProvider インターフェース）経由。
- アンダーライティング数学は純・決定論的な `lib/underwriting/`。LLMプロンプト内に置かない。
- `Finding` はバージョン付き zod スキーマ = AI と下流の硬い境界。生出力＋解析結果＋エンジンversion＋
  provider/model id を毎回永続し、全査定を replay/diff 可能にする。
